Factory Inventory Management

A full-stack demo with a Vue 3 single-page frontend, a FastAPI backend, and a JSON-file data layer loaded into memory at process start. This page summarises the tech stack, service topology, request lifecycle, and file layout.

Request lifecycle — dashboard example

End-to-end path for a single user navigating to / with a warehouse filter applied.

Route mount

User navigates to /. vue-router mounts Dashboard.vue; onMounted(loadData) fires.

Read filter state

getCurrentFilters() in useFilters.js reads the four module-level refs and returns {warehouse, category, status, month}. Because the refs are declared outside the factory function, every component sees the same singleton.

Parallel API calls

Promise.all dispatches getDashboardSummary, getOrders, getInventory, and getBacklog concurrently through the axios wrappers in api.js. Filters equal to 'all' are skipped when building URLSearchParams.

HTTP request

Browser issues GET http://localhost:8001/api/dashboard/summary?warehouse=San+Francisco. CORS middleware on the server accepts the origin.

Handler executes

get_dashboard_summary applies apply_filters(inventory_items, …) and filter_by_month(orders, month) over the in-memory lists loaded at import time, then computes aggregates (total_inventory_value, low_stock_items, …).

Pydantic & JSON

List endpoints validate against models like InventoryItem via response_model; the summary endpoint returns a plain dict. FastAPI serialises to JSON and responds with HTTP 200.

Refs update

axios resolves; responses are assigned to summary.value, allOrders.value, inventoryItems.value, allBacklogItems.value.

Computed & render

Derived computed properties (statusData, categoryData, orderTrendData, topProducts) recalculate. Vue's reactivity triggers re-render: KPI cards, donut chart, bar charts, and the backlog snapshot paint. A watch on the filter refs ensures any filter change re-runs loadData.

Known gaps & quirks

Worth flagging before extending the system.

Client/server contract drift

api.js exposes wrappers for /api/tasks and /api/purchase-orders, but server/main.py does not define those routes. Calls from App.vue at mount fail silently (console.error).

Orphan view

Backlog.vue is fully implemented but not registered in the router. Users reach backlog data only through the dashboard snapshot.

Reports ignore filters

/api/reports/quarterly and /api/reports/monthly-trends aggregate the full orders list irrespective of warehouse / category / status / month.

Two-source task state

App.vue merges locale-dependent mock tasks from currentUser.tasks with tasks fetched from the server, routing writes based on whether the id exists in the mock list.

Wide-open CORS

The FastAPI app sets allow_origins=["*"] with allow_credentials=True. Fine for a local demo; harden before shipping anywhere reachable.

Data loaded once at import

mock_data.py reads JSON at module import. Editing a JSON file requires restarting the server; there is no hot-reload of data.
